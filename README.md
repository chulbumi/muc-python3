## 사용시 주의사항

### Windows - GIT 클라이언트 사용
프로젝트에서 non-utf8포맷을 사용하므로, git 클라이언트에 따라 한글출력이 정상적이지 않을 수 있습니다.
* tortoiseGIT - 정상작동 확인
* 순정 git - diff tool 외 정상작동 확인

### 브랜치 Merge
TortoiseGit이 본 프로젝트 환경에 가장 안정적이나, 브랜치 병합(merge)시 다소 번거로움이 있습니다.
브랜치 병합 시에는 Github Desktop 사용 추천드립니다.(단, Diff는 정상적으로 보이지 않음)


## GIT 사용이 어색하신 분들을 위한 설명
 git은 저장소를 로컬과 원격의 양방향에 두고 쓰는 svn이라 생각하면 쉽습니다.
 그 외의 것들은 svn과 대동소이합니다. 

* commit을 하면 수정한 내용을 로컬 저장소에 올립니다.
* push를 하면 로컬저장소에 commit된 내용을 원격저장소로 올립니다
* pull을 하면 원격저장소에 업데이트된 내용을 로컬저장소로 내려받습니다.


## 빈 폴더 작성 시
개발을 하다보면 '빈 폴더'가 필요한 경우가 있습니다. 그런데 GIT은 빈 폴더는 커밋대상으로 취급하지 않습니다.
그래서 꼼수를 이용합니다.

빈폴더 안에 별도의 .gitignore를 등록하고 해당 파일 안에 이렇게 입력합니다

```
* //이 폴더 안의 모든 것을 무시
!.gitignore // 대신 이 파일 빼고
```

이렇게하면 빈 폴더가 Commit되고, 폴더 하위에 혹시라도 발생하는 Dummy Data는 커밋대상으로 취급하지 않습니다.
