# MUD 서버 실제 차이점 (발견된 버그)

## 1. 캐릭터 위치 로드 버그 (Rust 서버)

### 문제
Rust 서버가 캐릭터 파일에서 "위치" 정보를 로드하지만, 실제 게임 입장 시 **무시하고 항상 낙양성:1에 스폰**함

### 코드
```rust
// src/network/client.rs:1316
let _ = load_body_from_json(&mut player.body, &format!("data/user/{}.json", name));

// src/network/client.rs:1327 - 버그: 저장된 위치를 사용하지 않음
let start_pos = PlayerPosition::start(); // 항상 낙양성:1
```

### Python 서버 (정상 동작)
```python
# objs/player.py:1413-1421
rName = self.get('귀환지맵')
if rName == '':
    rName = '낙양성:42'
room = getRoom(rName)
# ... 저장된 위치 또는 귀환지를 사용
```

### 영향
- 캐릭터가 저장된 위치에 로그인되지 않음
- Python: "낙양성:42" (다른 방)
- Rust: "낙양성:1" (항상 같은 방)

---

## 2. 시작 방 불일치

### Python
- 기본 시작: "낙양성:42"
- 방 설명: "산들바람에 여유롭게 흔들리는 꽃들..."
- NPC: 왕대협

### Rust
- 기본 시작: "낙양성:1"
- 방 설명: "낙양성내의 가장 번화한 사거리..."
- NPC: 밍밍, 포졸

---

## 3. 명령어 응답 없음 (Python 서버)

### 문제
Python 서버에서 능력치, 무공, 소지품, 점수 등의 명령어 입력 시 결과가 표시되지 않음

### 가능한 원인
1. 명령어가 .py 파일과 .rhai 파일로 중복 정의됨
2. 스크립트 로드 순서 문제
3. 명령어 핸들러가 제대로 등록되지 않음

---

## 4. 공격 명령 동작 차이

### Python
- 결과: "☞ 무슨 말인지 모르겠어요. *^_^*"
- 공격 명령이 작동하지 않음 (방에 몹이 없음)

### Rust
- 결과: "포졸 을 공격했지만 빗나갔습니다!"
- 공격 명령이 정상 작동 (다른 방에 스폰됨)

---

## 수정 필요 항목

### 긴급 (Rust 서버)
1. **캐릭터 위치 로드**: 저장된 "위치" 필드를 읽어서 해당 위치에 스폰
2. **시작 방 통일**: Python과 동일한 시작 방 사용 (낙양성:42)

### 조사 필요 (Python 서버)
1. **명령어 응답 없음**: 능력치, 무공, 소지품, 점수 등이 왜 표시되지 않는지

### 테스트 필요
1. 전투 시스템 비교 (데미지 계산, 경험치 획득 등)
2. 아이템 획득/사용 비교
3. 스킬 사용 비교
