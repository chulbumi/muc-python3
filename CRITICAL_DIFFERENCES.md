# MUD 서버 간 심각한 차이점 발견

## 1. 시작 방/위치가 다름 (같은 캐릭터, 같은 저장 데이터)

### Python 서버 (9900)
- 방 이름: "낙양성"
- 방 설명: "산들바람에 여유롭게 흔들리는 꽃들..."
- NPC: 왕대협
- 출구: 서 (초보수련장)만 가능

### Rust 서버 (9999)
- 방 이름: "하남성낙양"
- 방 설명: "낙양성내의 가장 번화한 사거리..."
- NPC: 밍밍, 포졸
- 출구: 동, 아래, 북, 남, 서, 위

**문제점**: 캐릭터 저장 데이터에 "위치" 필드가 다름
- Python: "시작/시작" 또는 다른 값
- Rust: 다른 방으로 해석

---

## 2. 공격 명령 동작 차이

### Python 서버
```
공격 포졸
☞ 무슨 말인지 모르겠어요. *^_^*
```
- 공격 명령이 작동하지 않음
- 포졸이 방에 없음 (시작 방이 다름)

### Rust 서버
```
공격 포졸
포졸 을 공격했지만 빗나갔습니다!
검사 포졸 공격했지만 빗나갔습니다!
```
- 공격 명령이 정상 작동
- 포졸이 방에 있음

---

## 3. 명령어 응답 문제

### 능력치/무공/소지품/점수 명령

**Python 서버**: 명령을 입력해도 능력치/무공/소지품/점수 결과가 표시되지 않고
방 정보만 표시됨

**Rust 서버**: 동일하게 명령 결과가 표시되지 않음

**가능한 원인**:
1. 명령어가 스크립트(.rhai)로 구현되어 있지만 제대로 로드되지 않음
2. Python 서버는 .py 명령을 사용, Rust는 .rhai를 사용하는데 호환성 문제

---

## 4. 캐릭터 데이터 확인

```
"위치": "시작/시작"
```

두 서버가 "시작/시작"을 다르게 해석함:
- Python: 낙양성 (다른 방)
- Rust: 하남성낙양 (몹이 있는 방)

---

## 조사 필요 사항

1. **방 데이터 구조**: Python과 Rust의 방 데이터/파일이 다른지 확인
2. **명령어 매핑**: .py와 .rhai 명령이 제대로 매핑되는지 확인
3. **위치 시스템**: "시작/시작" 위치가 왜 다른 방으로 해석되는지 확인
4. **전투 시스템**: Python에서 공격 명령이 왜 작동하지 않는지 확인
