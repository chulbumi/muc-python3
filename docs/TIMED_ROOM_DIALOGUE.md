# 개인별 입장 안내 대화

몹의 `이벤트 $%입장이벤트%` Rhai 스크립트에서
`schedule_room_dialogue(초간격, 대사배열)`을 호출하면, 방에 들어온
각 사용자에게 독립적인 순서의 안내 대사를 보낼 수 있다.

```rhai
fn event() {
    schedule_room_dialogue(3, [
        "초보 안내인이 말합니다. \"먼저 주변을 잘 살피게.\"",
        "초보 안내인이 말합니다. \"서쪽의 초보수련장으로 가보게나.\"",
        "초보 안내인이 말합니다. \"막히면 도움말을 입력하면 된다네.\""
    ]);
}
```

위 예시는 입장 후 3초, 6초, 9초에 각각 한 줄을 출력한다. `[공]`과
`[사용자이름]` 표기는 기존 이벤트 출력과 같이 입장한 사용자 기준으로
치환된다.

각 예약은 접속 식별자, 입장 당시의 `존:방`, 그리고 그 한 번의 입장에만
쓰이는 임시 토큰을 함께 보관한다. 따라서 다른 사용자가 들어와도 자기
시퀀스를 처음부터 따로 받고, 사용자가 방을 나가거나 재접속하면 남은
대사는 출력되지 않는다. 같은 방에 다시 들어오면 입장이벤트가 새 토큰과
새 시퀀스를 만들므로 첫 줄부터 다시 시작한다. 재입장 횟수나 방별 진행도는
저장하지 않는다. 매 줄 출력 뒤에는 현재 기력/내공 프롬프트도 다시 표시한다.

몹 JSON에는 기존 입장이벤트와 같은 방식으로 Rhai 파일을 연결한다.

```json
"이벤트 $%입장이벤트%": "초보안내.rhai"
```

이벤트 파일은 다음 입장 시 다시 읽으므로, 관리자가 수정한 대사는 서버
재시작 없이 새로 시작되는 안내 시퀀스에 반영된다.
