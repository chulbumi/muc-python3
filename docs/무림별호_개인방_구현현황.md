# 무림별호 이벤트 및 개인방(사용자맵) 구현 현황

## 요약

- **무림별호.rhai**: Python의 입력 검증, 이벤트 조건, 중복 검사, 정/사파 설정,
  전체 공지, 개인방 생성·저장·이동 흐름을 구현.
- **개인방(사용자맵)**: Rust 귀환 명령이 `귀환지맵=사용자맵:이름` 형식 **지원** (PlayerPosition.room·parse_exits 문자열 지원 반영).
- **makeHome**: `world::user_home` + `make_home` efun으로 Python 방 JSON 구조 생성.
- **NICKNAME(닉네임 중복)**: `world::nickname`의 HashMap 레지스트리와
  `nickname_*` efun으로 `data/config/nickname.json` 호환 저장.
- **방 알림 대상**: 귀환 전·후 퇴장/진입 알림은
  `get_room_players(ob)`로 해당 방 사용자만 조회하고, 별호 전체 공지에만
  전체 접속자 목록을 사용한다.

---

## 1. 무림별호 이벤트 흐름 (Python 기준)

1. 왕대협(83) 대화로 `$이벤트설정 무림별호설정`, `$이벤트설정 무림별호 정파|사파` 설정
2. `[별호이름] 무림별호` 입력
3. 조건: `무림별호==''`, `checkEvent('무림별호설정')==true`, 3~10자, `NICKNAME.attr`에 미등록
4. `ob['무림별호']=line`, `ob['성격']=정파|사파`, `NICKNAME[line]=ob['이름']`, `NICKNAME.save()`
5. `ob.makeHome()` → `data/map/사용자맵/{이름}.map`(또는 JSON) 생성
6. `ob['귀환지맵']='사용자맵:{이름}'`, `ob.save()`
7. `getRoom('사용자맵:{이름}')` → `ob.enterRoom(room, '귀환', '귀환')` (개인방 입장)

## 3. 테스트 방법

### 3.1 무림별호.rhai (Rust)

- 전제: `무림별호설정` 이벤트와 `무림별호=정파|사파` 이벤트 값이 설정된 캐릭터.
- 입력: `[별호이름] 무림별호`.
- 기대: Python과 같은 길이·중복 검증 및 공지 후 개인방 JSON 생성,
  `무림별호/성격/귀환지맵` 저장, `사용자맵:{이름}` 입장.
- 단위 검증: 닉네임 레지스트리 저장 왕복, 생성한 개인방의 `RoomCache` 로드,
  Rhai 사용법·중복 분기, 같은 방/다른 방 알림 대상 분리.

### 3.2 귀환 + 사용자맵 (Rust, 수정 후) ✅

- 전제: `data/map/사용자맵/테스트.json` 존재, Body에 `귀환지맵=사용자맵:테스트` 설정
- 입력: `귀환`
- 기대: `사용자맵:테스트` 방으로 이동·표시.
- 구현: `PlayerPosition.room`을 `String`으로 변경, `return_command`에서 `zone:room`의 `room`을 문자열로 파싱, `Exit.destination`·`parse_exits`도 `(zone, room_id: String)` 지원.

### 3.4 Python (전체 흐름)

- 왕대협 대화로 `무림별호설정` 이벤트 획득 후
- `[별호] 무림별호` → `makeHome`·`귀환지맵`·`enterRoom` 까지 동작 확인.
- 이후 `귀환`으로 개인방 복귀 확인.
