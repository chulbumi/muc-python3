# `귀환` Python 동작 대조 기록

기준 코드는 `cmds/귀환.py`와 `objs/player.py`의
`enterRoom(room, '귀환', '귀환')` / `exitRoom('귀환', '귀환')`이다.
이 문서는 전체 이동 시스템의 완료 선언이 아니라, 현재 `귀환` 명령이 실제로
재현하는 범위와 안전하게 중단하는 범위를 구분한다.

## 현재 재현하는 순서

1. 현재 방 `귀환금지`를 `isMovable()`보다 먼저 검사한다.
2. 빈 `귀환지맵`만 `낙양성:42`로 대체한다. 비어 있지 않은 잘못된 값은
   기본값으로 바꾸지 않는다.
3. 레벨·힘·민첩 상한, 레벨 하한, 방 인원, 정/사파, 방파 출입 제한을 검사한다.
4. 보이는 사용자는 현재 방의 `이동스크립:귀환`을 사용한다. 스크립트가 없을
   때만 `무영지신` 기본 문구를 사용한다. `[공]`과 `postPosition1` 및 Python의
   사용자/방 수신자 CRLF 차이도 Rhai에서 처리한다.
5. 목적지의 안전하게 표현 가능한 `Room.update()`를 먼저 적용하고 위치를
   변경한 뒤 `viewMapData()` 결과를 출력한다.
6. 목적지 mob 삽입 순서로 `이벤트 $%입장이벤트%`를 실행한다. 현재 데이터에
   존재하는 입장이벤트는 `낙양성/이름맨`과 `선인/비학천룡` 두 Rhai 스크립트다.
7. 목적지 `진입스크립:귀환`이 있으면 그 문구만 방에 전송한다. 이 경우
   `하늘에서 ... '척~~~'` 기본 문구는 출력하지 않는다.
8. 첫 `체력감소` 속성은 피해 전 LP 프롬프트, 자신/같은 방 문구, HP 감소
   순서로 적용한다. 치명 피해도 이동을 완료한 뒤 보험 선택 드롭, 혼미,
   10단계 사망 heartbeat로 이어진다.
9. 모든 구방/신방 알림 대상은 `get_room_players(ob)`로 얻는다.

`Room.update()` 중 현재 재현하는 안전한 부분은 1초 갱신 제한과, 서 있고
체력·내공이 가득 차며 target/효과가 없는 mob의 Python `Body.tick` 증가다.
바닥 아이템 또는 아래의 관찰 가능한 mob 갱신 분기가 있으면 성공으로 처리하지
않는다.

## 이동 전에 중단하는 분기

아래 상태 코드는 Rhai에서 사용자 문구를 만들지 않으며 위치도 바꾸지 않는다.
Python 원본에 없는 오류 메시지를 추가하지 않기 위해서다.

| 상태 | Python에서 필요한 호출 | 현재 정확히 실행할 수 없는 이유 |
|---|---|---|
| `unsupported_room_item_update` | `Room.update()` → `Item.update()` / `timeofdrop` | Rust 방 바닥 저장에는 Python의 아이템별 `timeofdrop`이 없다. |
| `unsupported_mob_update` | `Room.update()` → `Mob.update()` | 사망/리젠, 아이템 리젠, 랜덤 대화·프롬프트, 활성 방어효과 등 해당 상태를 정확히 재현할 수 없는 경우다. |
| `unsupported_entry_event` | `doEvent()`의 `$엔터$`, `$위치이동`, `$스크립트호출` 또는 Rhai `wait_enter`/`set_position` | 귀환 처리 중 별도의 pending-input/중첩 이동/스크립트 상태를 네트워크에 원자적으로 넘길 수 없다. |

## 귀환에서 실행되지 않는 Python 분기

추종자는 별도 미구현 분기가 아니다. Python 조건은
`if f.env == prev and mode == '이동':`이고, `귀환.py`는 mode에 `'귀환'`을
전달하므로 이 루프의 본문은 원본에서도 실행되지 않는다. 이 조건은 회귀 테스트로
고정했다.

## 회귀 검증

`cargo test --lib return_home`은 다음을 실제 Rhai 명령을 통해 검증한다.

- Python 검사 순서와 잘못된 귀환지 처리
- 구방/신방만 받는 기본 귀환 알림
- 사용자 이동/진입 스크립트와 기본 문구 억제
- 방 보기 뒤 입장이벤트 실행
- 비치명 함정의 피해 전 프롬프트와 HP 감소
- 치명 함정의 이동 후 보험 드롭·혼미 상태와 자동이동 방의 주 이동 성공
- 귀환 mode에서 추종자 분기가 실행되지 않는 근거
