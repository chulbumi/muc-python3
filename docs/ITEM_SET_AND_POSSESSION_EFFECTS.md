# 아이템 세트 효과와 소지 효과

두 기능은 서로 독립적이다.

## 착용 세트 효과

세트 구성품은 `세트그룹`만 가지면 된다. 같은 인덱스의 아이템을 여러 개
착용해도 구성품 하나로 센다. 현재 착용(`inUse`) 중인 서로 다른 구성품
인덱스만 세트 수에 포함된다.

```json
{
  "세트그룹": "복면인세트"
}
```

세트가 발동하려면 현재 착용한 구성품 중 적어도 하나가 `세트조건`과
`세트효과`를 정의해야 한다.

```json
{
  "세트그룹": "복면인세트",
  "세트조건": 3,
  "세트효과": {
    "3": ["민첩 +100", "힘 -20"],
    "4": { "명중": 50 },
    "5": ["체력 +500", "방어력 +80"]
  }
}
```

조건을 만족한 단계는 누적된다. 위 세트를 네 개 착용하면 3단계와 4단계
효과가 함께 적용된다. 정의 아이템을 벗으면 나머지 구성품 수가 충분해도
세트 효과는 사라진다. 같은 그룹·단계를 여러 아이템이 중복 정의한 경우
아이템 인덱스 정렬상 첫 정의만 적용한다.

## 소지 효과

부적처럼 착용하지 않는 아이템은 별도 속성인 `소지효과`를 사용한다.
이 아이템은 세트 아이템일 필요가 없고 세트 수에도 포함되지 않는다.

```json
{
  "이름": "청룡부",
  "소지효과": ["운 +20", "마법발견 +10"]
}
```

동일 인덱스 수량이 여러 개여도 소지 효과는 한 번만 적용된다. 아이템을
전부 잃거나 소비하면 효과가 사라진다.

지원 능력치는 `공격력`, `방어력`, `힘`, `민첩성`(`민첩`), `맷집`,
`체력`, `내공`, `명중`, `회피`, `필살`, `운`, `경험치`, `마법발견`이다.
효과 값은 문자열 배열 또는 `{ "능력치": 수치 }` 객체로 작성할 수 있다.

아이템 JSON 변경은 서버 재시작 없이 다음 효과 재계산 시 반영된다.

## 소모품 시간제 효과

소모품의 `사용효과`는 세트 효과 및 소지 효과와 별도의 시간제 버프다.
`[아이템] 먹어` 또는 `[아이템] 사용`이 성공하여 아이템 한 개를 소비한
뒤 발동한다. 동일한 `사용` 아이템 이벤트가 있으면 기존 원칙대로 이벤트가
시스템 명령보다 먼저 실행된다.

```json
{
  "이름": "호심단",
  "종류": "먹는것",
  "사용스크립": "$아이템$을 삼킵니다.",
  "사용효과": {
    "이름": "호심진기",
    "지속시간": 60,
    "효과": {
      "맷집": 30,
      "방어력": 50,
      "힘": -5
    },
    "발동스크립": "$효과$가 $지속시간$초 동안 온몸을 감쌉니다.",
    "만료스크립": "호심진기가 안개처럼 흩어집니다."
  }
}
```

- `지속시간` 단위는 실제 시간의 초다.
- `효과` 대신 `능력치`라는 키도 사용할 수 있다.
- 지원 능력치는 세트·소지 효과와 같다.
- 같은 아이템을 다시 소비하면 효과 수치는 중첩되지 않고 제한시간만
  처음부터 갱신된다.
- 서로 다른 아이템의 시간제 효과는 함께 적용된다.
- 활성 효과와 만료 시각은 캐릭터 속성 `소모품버프`에 저장된다. 로그아웃
  중에도 시간이 흐르므로 재접속으로 지속시간을 초기화할 수 없다.
- `발동스크립`의 `$아이템$`, `$효과$`, `$지속시간$`은 Rhai 명령에서
  치환된다. `만료스크립`도 Rhai 전투 틱 출력 경로에서 표시된다.

## 복용 시 영구 특성 증가

`영구효과`는 이벤트가 아니다. `[아이템] 먹어` 또는 `[아이템] 사용`이
아이템 한 개를 정상적으로 소비한 직후 공통 소비 처리에서 직접 적용하고
캐릭터를 저장한다. 같은 아이템을 반복 복용하면 매번 다시 증가한다.

```json
{
  "영구효과": {
    "효과": {
      "힘": 1,
      "민첩성": 2,
      "맷집": 1,
      "명중": 1,
      "회피": 1,
      "필살": 1,
      "운": 1,
      "내공": 10,
      "체력": 100
    },
    "발동스크립": "$영구효과$의 공력이 몸에 새겨집니다."
  }
}
```

`내공`은 `최고내공`, `체력`은 `최고체력`을 증가시킨다. 나머지는 해당
기본 특성치를 증가시킨다. 증가량은 양수만 적용한다. `발동스크립`을
생략하면 Rhai 명령이 증가한 항목과 수치를 기본 문구로 출력한다.

자유 특성치의 `올려`와 `내려`는 `명중`, `회피`, `필살`, `운` 네 항목만
허용한다. 영구 복용으로 증가한 이 네 수치는 자유 특성치 배분 기록에
더하지 않으므로 `내려`로 회수할 수 없다. 힘·민첩성·맷집·내공·체력은
`올려`와 `내려` 명령으로 변경할 수 없다.
