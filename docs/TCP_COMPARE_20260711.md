# Python ↔ Rust TCP 비교 (2026-07-11)

기존 캐릭터 `비교테스터`와 저장된 암호 `test1234`로 Python `9903`, Rust
`9999`를 각각 기동한 뒤 동일한 명령을 raw socket으로 보냈다. ANSI를 제거한
응답의 바이트 수는 다음과 같다.

| 명령 | Python | Rust | 관찰 |
|---|---:|---:|---|
| 능력치 | 274 | 293 | Rust 응답에 로그인 직후 입장 안내가 남아 있음 |
| 소지품 | 212 | 210 | 레이아웃/줄바꿈 차이 |
| 봐 | 241 | 240 | 레이아웃/줄바꿈 차이 |
| 저장 | 40 | 38 | prompt 줄바꿈 차이 |

두 서버 모두 기동·로그인·명령 실행에 성공했다. 바이트 수가 같지 않으므로
이 결과를 호환성 완료 증거로 사용하지 않는다. 다음 비교에서는 로그인 응답을
완전히 소진한 뒤 명령별 raw CRLF를 저장해 정확한 문장·레이아웃 차이를 고친다.

## 자동 quick 재검증

최신 서버를 10초간 기동해 `mud-test.py quick --py-port=9903 --rust-port=9999`를 실행했으나, 비교용 Python 계정의 저장 데이터가 손상되어 Python 원본에서 `최고체력` 문자열을 정수로 나누는 `TypeError`가 발생했다. 이는 Rust 응답 불일치가 아니라 비교 계정 데이터 전처리 실패이므로 통과 증거로 사용하지 않는다.

정상 계정 `비교테스터`로 로그인 후 초기 출력을 비우고 다시 비교한 결과는 Python/Rust 각각 `능력치` 653/713, `소지품` 507/505, `봐` 606/599, `저장` 64/62 바이트였다. `능력치` Rust 응답에는 로그인 입장 안내가 남아 있어 수신 타이밍 차이가 확인되었고, 나머지는 CRLF·prompt 레이아웃 차이를 추가 대조해야 한다.
