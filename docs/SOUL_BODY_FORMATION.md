# Soul / Body 캐릭터 전환과 무리 코어

## 경계

- `Soul`: 한 접속자가 소유한 캐릭터 목록, 주 캐릭터, 현재 조종 캐릭터,
  무리 구성 순서를 관리합니다.
- `Body`: 이름, 능력치, 아이템, 무공, 전투 상태처럼 캐릭터마다 달라지는
  실제 게임 데이터를 관리합니다. `Body`는 코드 내부 명칭이며 게임 문구에는
  노출하지 않습니다.
- `Client`와 명령 입력 상태는 유지한 채 활성 `Body`만 교체하므로
  `1 → 2`, `2 → 3`, `3 → 1`, `1 → 3` 전환을 모두 지원합니다.

주 캐릭터를 포함해 Soul 하나당 최대 4명입니다. 역할은 `main`, `alternate`,
`mercenary`이며 사용자에게는 주 캐릭터, 보조 캐릭터, 용병으로 표시합니다.
목록은 `data/soul/<주캐릭터>.json`에 저장하고 각 캐릭터 데이터는 기존처럼
`data/user/<캐릭터>.json`에 독립 저장합니다.

## 게임 동작

- `전환`: 목록 표시
- `[번호|이름] 전환`: 같은 Soul의 캐릭터로 조종 대상을 변경
- 비활성 캐릭터는 소켓 없는 사용자 객체로 방에 실제 배치됩니다.
- 조종 캐릭터가 정상 이동하면 모든 동행 캐릭터를 같은 목적지로 옮깁니다.
- 정상 명령 이동 직후에는 전원을 조종 캐릭터의 목적지로 옮기고, 매 게임 틱의
  이탈 복구 기준은 주 캐릭터 위치로 삼습니다. 따라서 향후 용병·보조 캐릭터의
  랜덤이동이나 자동이동이 추가되어도 주 캐릭터가 있는 무리로 복귀합니다.
- 조종 캐릭터가 몹 전투에 들어가면 같은 Soul의 생존 동행 캐릭터도 같은 몹
  인스턴스를 목표로 전투 틱을 수행합니다. 피해량은 각 캐릭터 이름으로 따로
  기록되며 경험치 기여도 계산에 사용됩니다.
- 서버 스크립트는 `request_soul_attach(ob, 이름, 역할)`로 보조 캐릭터나
  용병을 Soul에 편입할 수 있습니다. 편입 대상은 기존 사용자 JSON으로부터
  로드하며, 이미 접속 중이거나 정원 4명을 넘으면 거부합니다.
- `request_soul_switch()`는 실제 네트워크 상태 변경을 요청할 뿐이고, 출력은
  `cmds/전환.rhai`가 담당합니다.

기존 관리자 `체인지`는 서로 다른 접속자의 Player/Soul 전체를 교환하는 별도
기능으로 유지합니다. 일반 사용자의 소유 캐릭터 전환은 `전환`을 사용합니다.

## 파티와 경험치

일반 파티도 대장 포함 최대 4명입니다. Soul 동행 캐릭터와 외부 파티원을 합친
전투 무리 전체가 4명을 넘을 수 없습니다. 몹 경험치는 같은 방에 있고 실제 피해를
준 최대 4명에게만 배분합니다.

분배 가중치는 피해 기여도를 주축으로 하며 다음 패널티를 곱합니다.

- 파티 최고 레벨과의 격차가 클수록 제곱 비율로 감소
- 자신의 레벨에 비해 몹 레벨이 지나치게 높으면 추가 감소
- 총 피해의 5% 미만이면 기여도 패널티 추가

비슷한 레벨이고 각자 10% 이상 기여한 구성원은 한 명이 늘 때마다 전체 경험치
풀에 10% 협동 보너스를 받으며, 최대 30%입니다. 따라서 정상적인 2~4인 파티는
손해가 완화되지만, 고레벨이 대부분의 피해를 담당하고 저레벨이 한 번만 공격한
경우 저레벨 보상은 극소량으로 제한됩니다.
