# 포장(묶음) 아이템

`data/item/<포장인덱스>.json`의 `아이템정보`에 다음 속성을 지정하면 동일한
원본 아이템을 한 칸으로 판매·보관할 수 있습니다.

```json
{
  "아이템정보": {
    "이름": "생수 100개 포장",
    "종류": "포장",
    "포장원본": "생수",
    "포장수량": 100,
    "무게": 100
  }
}
```

`포장원본`은 표시 이름이 아니라 아이템 JSON 인덱스입니다. 포장 아이템도
일반 아이템처럼 `값`, `반응이름`, 상점 정보와 이벤트를 가질 수 있으므로
상점에서 포장 단위로 구입할 수 있습니다. 구매·저장 시 포장 하나는
`inv_stack`의 수량 1개이며, 저장 JSON도 `{인덱스, 수량}` 형식을 그대로
사용합니다.

사용자는 `생수 100개 포장 풀어`처럼 입력합니다. 시스템 명령보다 먼저
아이템 이벤트를 조사하므로, `밧줄 풀어`에 밧줄 이벤트가 있으면 그 이벤트가
먼저 실행되고 포장 해제 명령은 실행되지 않습니다.

원본은 반복 가능한 비장비 아이템이어야 합니다. `단일아이템`, 개별 인스턴스,
UUID·강화·확장 속성이 있거나 무기·방어구·호위인 원본은 중복을 만들 수 없으므로
포장 해제를 거부합니다. `묶음`과 `꾸러미`도 입력 호환을 위해 허용하지만 새
정의에는 한자어 기반 명칭인 `포장`을 권장합니다.
