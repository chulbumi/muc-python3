# 런타임 월드 편집 코어

관리자와 신뢰된 게임 이벤트가 서버 재시작 없이 월드를 확장할 수 있도록, 파일 저장과 현재 프로세스의 캐시 갱신을 하나의 작업으로 묶는다. 화면에 보이는 문구는 `cmds/월드편집.rhai` 또는 각 이벤트 Rhai가 담당한다.

## 관리자 명령

형식은 `[작업] [JSON] 월드편집`이며 관리자 등급 2000이 필요하다.

```text
방 {"zone":"시험존","room":"1","attrs":{"이름":"청죽림 별원","설명":["대숲 사이에 작은 별원이 있다."],"출구":[]}} 월드편집
출구 {"zone":"시험존","room":"1","name":"안쪽","destination_zone":"시험존","destination_room":"2","hidden":false} 월드편집
몹 {"zone":"시험존","key":"청의검객","attrs":{"이름":"청의검객","체력":120,"레벨":10}} 월드편집
몹배치 {"zone":"시험존","room":"1","mob_key":"시험존:청의검객"} 월드편집
아이템 {"key":"비취비수","attrs":{"이름":"비취비수","종류":"무기","설명1":"비취빛 비수가 놓여 있다."}} 월드편집
고정물 {"zone":"시험존","room":"1","key":"벽화","kind":"mechanism","attrs":{"name":"낡은 벽화","설명1":"낡은 벽화가 걸려 있다.","events":{"밀어":"벽화_밀어.rhai"}}} 월드편집
이벤트 {"zone":"시험존","path":"벽화_밀어.rhai","source":"fn main(ob, fixture_id, cmdline) { send_line(ob, \"벽화 뒤에서 바람이 샌다.\"); }"} 월드편집
```

작업은 허용된 `data/map`, `data/mob`, `data/item`, `data/script` 아래에만 원자적으로 저장된다. `..`, 절대 경로, 경로 구분자를 이용한 이탈은 거부한다. 이벤트는 새 파일을 교체하기 전에 Rhai 문법 검사를 통과해야 한다.

## 즉시 반영 범위

- 방·출구·방 이벤트·Fixture 선언: 기존 방 `Arc`를 유지하면서 정적 정의만 다시 읽는다. 현재 플레이어와 바닥 상태는 보존한다.
- 몹 정의: 템플릿만 다시 읽으며 이미 생성된 몹의 현재 체력과 전투 상태는 바꾸지 않는다.
- 몹 배치: 방 선언에 영구 저장하고 현재 방에도 한 개체를 즉시 생성한다.
- 아이템 정의: `ItemCache` 템플릿을 즉시 교체한다.
- Fixture: 방 안의 `placement_key`가 같으면 runtime id를 유지하면서 종류·속성·이벤트를 교체한다.
- Fixture/아이템 이벤트: 평소에는 캐시된 AST를 실행한다. 파일의 수정시각·크기·파일 identity가 바뀐 경우에만 다시 읽고 컴파일한다. 관리자 이벤트 편집은 원자 교체이므로 서버 재시작 없이 다음 트리거부터 새 AST가 실행된다.

## 양도 불가 월드 도구 아이템

일반 사용자에게 전역 편집 권한을 주지 않는다. 관리자가 만든 아이템 정의에 `아이템속성`의 양도 불가 플래그와 `월드권한`을 함께 넣고, 그 아이템의 이벤트가 실행되는 동안에만 `item_world_edit`를 제공한다. 선택된 인벤토리 아이템 키는 Rust가 캡처하므로 이벤트가 더 강한 다른 아이템을 사칭할 수 없다.

```json
{
  "이름": "천공개물 잔권",
  "종류": "서책",
  "아이템속성": ["줄수없음", "버리지못함"],
  "월드권한": ["home.room", "home.exit", "home.fixture"],
  "events": {"사용": "천공개물_사용.rhai"}
}
```

아이템 이벤트에서는 다음처럼 사용한다.

```rhai
let result = item_world_edit(ob, "방", #{
    room: ob["이름"] + "_비밀방",
    attrs: #{ "이름": "기관 별채", "설명": ["기관패가 펼쳐 만든 별채다."] }
});
```

권한과 작업 대응은 다음과 같다.

- `home.room`: 개인 방 생성·편집
- `home.exit`: 소유 방의 출구 생성·연결
- `home.fixture`: 소유 방의 Fixture 생성·편집
- `home.mob.author`: 사용자 전용 몹 정의 생성·편집
- `home.mob.place`: 사용자 전용 몹을 소유 방에 영구 배치

대상 방은 `사용자맵`에 있고 `주인`이 실행 사용자와 같아야 한다. 새 확장 방은 충돌 방지를 위해 `사용자이름_...` 영역에 생성된다. 사용자 몹의 이름은 자유롭게 정할 수 있지만 항상 `사용자맵`에 저장되고 `주인`은 코어가 강제로 기록된다. 이미 존재하는 몹은 같은 `주인`만 편집할 수 있으며, `home.mob.place`도 정의의 `주인`이 실행 사용자와 같은 몹만 허용한다. 아이템에 `줄수없음`, `양도불가`, `거래불가` 중 하나가 없으면 모든 월드 capability를 거부한다.

따라서 `천공개물` 같은 개인 건축 서책에는 방·출구·Fixture 권한만 줄 수 있다. 일반 아이템 이벤트는 전역 몹·아이템 정의나 이벤트 소스 자체를 작성할 수 없다. 관리자는 `world_edit`/`world_edit_json`으로 모든 존을 편집할 수 있다.

## 존 위임형 크리에이터 서책

일반 사용자를 특정 존의 크리에이터로 임명할 때는 서책에 `zone.creator`와 허용 존을 함께 기록한다.

```json
{
  "이름": "손자병법 위임본",
  "종류": "서책",
  "아이템속성": ["양도불가"],
  "월드권한": ["zone.creator"],
  "월드존": ["북방전장", "북방전장심층"],
  "events": {"사용": "손자병법_사용.rhai"}
}
```

`zone.creator`는 지정된 존 안에서 `방`, `출구`, `몹`, `몹배치`, `고정물`, `이벤트` 작업을 허용한다. 방과 몹 이름에는 사용자 접두사 제약이 없으며, 기존 정의도 자유롭게 편집할 수 있다. 대신 다음 경계는 코어가 강제한다.

- 모든 작업의 `zone`은 서책의 `월드존` 목록 안에 있어야 한다.
- 출구의 목적지 존도 같은 위임 목록 안에 있어야 한다.
- 배치할 몹의 템플릿 존과 배치 대상 존 모두 위임 목록 안에 있어야 한다.
- 이벤트 소스도 `data/script/<위임존>` 아래에서만 작성·교체할 수 있다.
- 아이템 정의는 존 소속이 아니므로 `zone.creator`로 제작·편집할 수 없다.
- 서책이 양도 가능해지면 모든 위임 권한은 즉시 거부된다.

필요하면 `zone.creator` 대신 `zone.room`, `zone.exit`, `zone.fixture`, `zone.mob.author`, `zone.mob.place`, `zone.event`를 조합해 더 좁게 위임할 수도 있다. 같은 아이템 이벤트 API인 `item_world_edit`를 사용하며, 다음 트리거부터 수정된 이벤트 AST가 재시작 없이 적용된다.

Fixture나 서버가 신뢰하는 비아이템 이벤트가 개인방을 확장할 필요가 있을 때는 `home_world_edit`를 사용할 수 있지만, 아이템 이벤트 엔진에는 이 우회 API를 등록하지 않는다.

방, 아이템, Fixture의 `attrs`에는 `events` 또는 `이벤트스크립트` 맵을 넣어 임의 트리거와 Rhai 파일을 연결할 수 있다. 세부 게임 규칙과 출력은 코어에 추가하지 않고 이 이벤트 스크립트에서 확장한다.
