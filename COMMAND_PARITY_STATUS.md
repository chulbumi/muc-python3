# Python → Rust/Rhai 명령 비교 현황

최종 갱신: 2026-07-13

## 완료 판정 기준

명령을 완료로 표시하려면 다음을 모두 확인해야 한다.

1. Python 원본의 사용법, 권한, 실패 분기와 성공 조건을 직접 읽음
2. 필요한 방·대상·아이템·이벤트·전투·무리 상태를 실제로 구성함
3. 본인, 대상, 같은 방 목격자의 출력 바이트(개행, ANSI, 조사, 프롬프트)를 비교함
4. 체력·내공·은전·소지품·방·전투·무리 등 상태 변화를 비교함
5. 해당 성립 조건을 재현하는 회귀 테스트가 있고 전체 `cargo test --lib` 통과

파일이 존재하거나 단순 응답이 나오는 것만으로는 완료로 보지 않는다. 추가 차이가 발견되면 완료 명령도 다시 미완료로 돌린다.

## 범위

- Python과 Rhai가 모두 있고 Python `CmdObj`가 있는 플레이어 명령: **189개**
- 현재 완료: **189개**
- 현재 해야 할 것: **0개**
- `comm.py`/`comm.rhai`는 `CmdObj`가 없는 follower/group helper이며 명령 등록기에서도 비공개로 제외되므로 플레이어 명령 집계에서 제외
- Rhai에만 있는 16개도 검토 완료: `attack`, `debug_test`, `help`, `inventory`, `look`, `master`, `say`, `test`, `test_output`, `test_simple`, `test_syntax`, `디버그`, `주다`, `테스트명령`은 등록 차단된 호환·개발·중복 소스이며, `끝`·`종료`만 Python `Player.parse_command`와 같은 비공개 registry hook으로 등록된다.
