# Python → Rust/Rhai 명령 비교 현황

최종 갱신: 2026-07-14

## 완료 판정 기준

명령을 완료로 표시하려면 다음을 모두 확인해야 한다.

1. Python 원본의 사용법, 권한, 실패 분기와 성공 조건을 직접 읽음
2. 필요한 방·대상·아이템·이벤트·전투·무리 상태를 실제로 구성함
3. 본인, 대상, 같은 방 목격자의 출력 바이트(개행, ANSI, 조사, 프롬프트)를 비교함
4. 체력·내공·은전·소지품·방·전투·무리 등 상태 변화를 비교함
5. 해당 성립 조건을 재현하는 회귀 테스트가 있고 전체 `cargo test --lib` 통과

파일이 존재하거나 단순 응답이 나오는 것만으로는 완료로 보지 않는다. 추가 차이가 발견되면 완료 명령도 다시 미완료로 돌린다.

     `$몹상태설정 리젠`이 Rhai 주석으로만 남아 있던 것을 실제 efun 호출로 복원했다.
     임자영의 6단계 시작부터 수령봉→검주봉→삼화봉→양정봉, 주명명머리 전달,
     대렴봉 완료·188번 이동, 소설령의 환룡석·만년화리의내단 보상까지 원본 순서와
     관련 맵 배치를 회귀 검증해 완료로 올렸다.
   - `해적선`: 원본 데이터에는 이 분류명과 같은 독립 NPC·보상 문자열이 없다. 현재
     확인 가능한 동정호 뱃사공→월환도 항로는 강태공 의뢰, 흑괴머리(살아 있으면 전투,
     시체면 머리 지급·리젠), 초선의 손수건, 뱃사공의 `동정호:200` 이동 및 200→206
     월환도 접근 출구까지 회귀 검증했다. 이 항로가 도움말의 해적선 분류 전체인지
     원본 공략 또는 추가 식별자로 재확인하기 전까지 완료로 올리지 않는다.
   - `혈살루·보타암`: 검후의 용천검·검후의눈물 단일 기연 대체, 적수염마의 시체
     전이, 검후/적수염마 선택 분기의 체력 200,000 처벌을 복원하고 회귀 검증했다.

     200,000 체력 감소가 되도록 `has_skill` 이벤트 efun과 함께 복원했다.
     검후의 역분기 `적수염마무기`도 천인혈주(진품 또는 가품)와 `혈세천하`를 모두
